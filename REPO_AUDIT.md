# Repo Audit

This audit summarizes the cleanup and presentation work completed for the recruiter-facing portfolio. Strategy logic, selected parameters, cost assumptions, and saved headline metrics were not changed during the final readiness pass.

## Current Structure

- `hft_microstructure/`: C++ HFT-inspired microstructure simulator with saved historical-style SPY results, deterministic synthetic demo diagnostics, retained trade logs, rejected-signal logs, ablations, proxy adverse-selection stress, and regenerated plots.
- `medium_term_alpha/`: Python cross-sectional alpha research project with saved selected-default metrics, benchmark comparison, walk-forward/sensitivity/capacity diagnostics, sample-run audit exports, and regenerated plots.

## Removed Earlier

- Placeholder/template-only HFT ablation and latency CSVs.
- Placeholder medium-term trade-log sample.
- Duplicate SVG plots that were replaced by reproducible PNG plots.
- Build artifacts, temporary outputs, cache folders, and scratch validation files where removable.

## Evidence Boundaries

- HFT 30-session headline results are saved historical-style SPY quote-replay outputs; full raw quote files are excluded due to size.
- The final micro-alpha research-quality scorecard supports a 9/10 research-portfolio rating, not a live-trading rating.
- HFT `trade_log.csv`, `rejected_signals.csv`, `ablation_results.csv`, and `latency_sensitivity.csv` are real simulator outputs from a deterministic synthetic demo stream, not historical performance evidence.
- HFT latency sensitivity is proxy adverse-selection stress, not true timestamp-shift latency modelling.
- Medium-term headline metrics are selected-default evidence from saved result CSVs.
- Medium-term holdings and benchmark time-series audit CSVs are from the committed fixed sample run, not the full selected-default online run.

## Kept

- Reviewer-facing result CSVs and plots under each project’s `Results/` and `Plots/` folders.
- Small sample datasets for smoke testing and audit workflow reproduction.
- Documentation files that explain methodology, limitations, validation, and run commands.

## Still Needs Improvement

- Regenerate HFT trade/fill logs, ablations, and adverse-selection stress on the full historical quote dataset.
- Add true latency modelling only if timestamped order/fill data supports it.
- Retain full selected-default medium-term holdings from a pinned market-data snapshot.
- Replace survivorship-biased medium-term universe with point-in-time, delisting-aware data.
- Calibrate trading costs, fills, slippage, and market impact from execution data.
